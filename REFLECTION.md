Reflection

For this project, I chose to work with my Spotify liked songs data. I thought it would be interesting to analyze something personal and see patterns in the music I listen to. The dataset contained many columns and some messy values, so part of the project was cleaning the data and selecting the most useful variables.

One thing I learned from this project was how semi-structured data can still require a lot of cleaning before it becomes useful for analysis. Even though the data came from Spotify, some columns needed to be converted into proper formats, and some values were missing or inconsistent. Using pandas made it easier to explore the dataset and transform it into a cleaner version.

Working with data in GitHub was also a useful experience. I learned how to organize a repository with folders for raw data, cleaned data, notebooks, and scripts. Making multiple commits with clear messages also helped track the changes made during the project.

I did use AI tools occasionally during the process. AI helped me understand some pandas functions and helped me debug errors in the code. However, I still needed to review the suggestions carefully because sometimes the explanations were not fully clear or needed adjustments to work with my dataset.

One challenge I experienced was understanding how to structure the project so it would make sense to someone else reading the repository. Organizing folders, writing documentation, and making the project readable took more effort than I expected, but it also made the final project look more professional.
